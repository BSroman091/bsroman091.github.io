1. ¿Qué es Java?

Java es un lenguaje de programación de alto nivel, orientado a objetos y ampliamente utilizado en el desarrollo de software moderno. Fue creado por James Gosling en 1995 con el objetivo de diseñar un lenguaje que fuera portátil, seguro y fácil de usar en diferentes plataformas. Desde entonces, Java se ha convertido en uno de los lenguajes más importantes del mundo, especialmente en entornos empresariales.

Una de sus características más destacadas es su capacidad de ejecutarse en múltiples sistemas operativos sin necesidad de modificar el código. Esto es posible gracias a la Máquina Virtual de Java (JVM), que interpreta el código compilado y lo adapta al sistema donde se ejecuta. Este enfoque permite que un programa desarrollado en Windows pueda ejecutarse en Linux o Mac sin cambios.

Java se utiliza en una gran variedad de aplicaciones, incluyendo desarrollo de aplicaciones móviles (especialmente Android), sistemas bancarios, plataformas web, software empresarial y aplicaciones científicas. Su estabilidad y seguridad lo hacen ideal para proyectos grandes y complejos.

Además, Java cuenta con una enorme comunidad de desarrolladores y una gran cantidad de documentación, lo que facilita el aprendizaje y la resolución de problemas.

public class Main {

Características Principales de Java

  • Orientado a Objetos: Todo se basa en clases y objetos
  • Portátil: Funciona en cualquier plataforma con JVM
  • Seguro: Manejo de errores y gestión de memoria
  • Compilado e Interpretado: Se compila a bytecode y se interpreta en JVM
  • Robusto: Ideal para aplicaciones grandes y críticas

Componentes Principales

Componente Descripción
JDK (Java Development Kit) Herramientas para desarrollar aplicaciones Java
JRE (Java Runtime Environment) Entorno necesario para ejecutar aplicaciones Java
JVM (Java Virtual Machine) Máquina virtual que ejecuta el bytecode
Dato Histórico

Java fue creado originalmente para dispositivos televisivos, pero resultó ser mucho más versátil. Su lema "Write Once, Run Anywhere" (WORA) refleja su capacidad de portabilidad.

2. Sintaxis Estructurada y Uso de Clases

Java tiene una sintaxis más estructurada y estricta en comparación con otros lenguajes como Python. Todo el código en Java debe estar contenido dentro de clases, lo que refleja su enfoque orientado a objetos desde el inicio.

Cada programa en Java comienza con una clase principal que contiene el método main, el cual es el punto de entrada del programa. Este método es donde se ejecutan las instrucciones iniciales. La sintaxis incluye el uso de llaves {} para definir bloques de código y punto y coma ; para terminar instrucciones.

Aunque esta estructura puede parecer más compleja al principio, ayuda a mantener el código organizado y reduce errores en programas grandes. También fomenta buenas prácticas desde el inicio del aprendizaje.

Estructura Básica

public class Saludo {
    public static void main(String[] args) {
        System.out.println("Bienvenido a Java");
    }
}

Análisis de la Estructura

  • public class Saludo → define la clase pública llamada "Saludo"
  • public static void main(String[] args) → método principal donde comienza la ejecución
  • System.out.println() → imprime un mensaje en pantalla
  • Las llaves {} delimitan los bloques de código
  • Los puntos y comas ; terminan cada instrucción

Comparación con Python

Aspecto Java Python
Estructura Obligatoriamente en clases Flexible, puede ser procedural
Delimitadores de bloque Llaves { } Indentación
Terminador de instrucción Punto y coma ; Salto de línea

Complejidad inicial Mayor Menor

static = no necesita instancia

3. Variables y Tipos de Datos

En Java, las variables se utilizan para almacenar información que puede cambiar durante la ejecución del programa. A diferencia de lenguajes dinámicos, Java requiere que cada variable tenga un tipo de dato definido desde el inicio, lo que se conoce como tipado estático.

Esto significa que antes de usar una variable, debes especificar si será un número, texto, carácter, etc. Este enfoque ayuda a prevenir errores y hace que el código sea más seguro y predecible.

Tipos de Datos Primitivos

int (Enteros): Números sin decimales (-2,147,483,648 a 2,147,483,647) 
int edad = 20;
int cantidad = 100;
double (Decimales): Números con decimales de precisión 
double altura = 1.75;
double precio = 19.99;
char (Carácter): Un único carácter 
char letra = 'A';
char digito = '5';
boolean (Booleano): Verdadero o falso 
boolean activo = true;
boolean disponible = false;
String (Texto): Cadena de caracteres 
String nombre = "Luis";
String ciudad = "Madrid";

Ejemplo Completo

public class Variables {
    public static void main(String[] args) {
        int edad = 20;
        double altura = 1.75;
        String nombre = "Luis";
        boolean activo = true;

        System.out.println(nombre + " tiene " + edad + " años");
        System.out.println("Altura: " + altura + " metros");
        System.out.println("Activo: " + activo);
    }
}

Operaciones Básicas

int a = 10;
int b = 3;

int suma = a + b;          // 13
int resta = a - b;         // 7
int multiplicacion = a * b; // 30
int division = a / b;      // 3
int modulo = a % b;        // 1
int potencia = a;          // No existe operador directo

Conversión de Tipos

// Conversión automática
int numero = 10;
double decimal = numero;   // Conversión implícita

// Conversión forzada
double decimal2 = 10.5;
int numero2 = (int) decimal2;  // Conversión explícita (casting)
Tipado Estático vs Dinámico

En Java (tipado estático) debes especificar el tipo: int numero = 5;
En Python (tipado dinámico) no lo necesitas: numero = 5

4. Estructuras de Control

Las estructuras de control permiten dirigir el flujo de un programa, tomando decisiones o repitiendo acciones. Son fundamentales para crear programas interactivos y funcionales.

Condicionales (if-else)

Las estructuras condicionales permiten ejecutar diferentes bloques de código dependiendo de una condición.

Ejemplo Básico

int numero = 3;

if (numero % 2 == 0) {
    System.out.println("Es par");
} else {
    System.out.println("Es impar");
}

Ejemplo con else if

int edad = 15;

if (edad >= 18) {
    System.out.println("Mayor de edad");
} else if (edad >= 13) {
    System.out.println("Adolescente");
} else {
    System.out.println("Niño");
}

Switch

Para múltiples opciones basadas en un valor específico:

int dia = 3;

switch (dia) {
    case 1:
        System.out.println("Lunes");
        break;
    case 2:
        System.out.println("Martes");
        break;
    case 3:
        System.out.println("Miércoles");
        break;
    default:
        System.out.println("Otro día");
}

Bucle for

for (int i = 1; i <= 5; i++) {
    System.out.println("Número: " + i);
}

Bucle while

int contador = 1;

while (contador <= 5) {
    System.out.println("Contador: " + contador);
    contador++;
}

Bucle do-while

Se ejecuta al menos una vez, luego verifica la condición:

int contador = 1;

do {
    System.out.println("Contador: " + contador);
    contador++;
} while (contador <= 5);

5. Métodos (Funciones)

En Java, los métodos son bloques de código que realizan una tarea específica y pueden ser reutilizados varias veces dentro de un programa. Esto ayuda a evitar la repetición de código y facilita la organización.

Cada método tiene un tipo de retorno (el tipo de dato que devuelve), un nombre y opcionalmente parámetros. Si no devuelve nada, se usa void.

Estructura de un Método

[modificador] [tipo_retorno] nombreMetodo([parámetros]) {
    // Cuerpo del método
    return valor;
}

Método que Multiplica

public class Operaciones {

    public static int multiplicar(int a, int b) {
        return a * b;
    }

    public static void main(String[] args) {
        int resultado = multiplicar(4, 5);
        System.out.println("Resultado: " + resultado);  // 20
    }
}

Métodos sin Retorno (void)

public class Saludos {

    public static void saludar(String nombre) {
        System.out.println("Hola, " + nombre);
    }

    public static void main(String[] args) {
        saludar("María");   // Hola, María
        saludar("Juan");    // Hola, Juan
    }
}

Método que Devuelve Múltiples Cálculos

public static double calcularPromedio(double a, double b, double c) {
    return (a + b + c) / 3;
}

public static void main(String[] args) {
    double promedio = calcularPromedio(8.5, 9.0, 7.5);
    System.out.println("Promedio: " + promedio);
}

Sobrecarga de Métodos

Java permite tener múltiples métodos con el mismo nombre pero diferentes parámetros:

public static int sumar(int a, int b) {
    return a + b;
}

public static double sumar(double a, double b) {
    return a + b;
}

// El compilador elige automáticamente el método correcto
System.out.println(sumar(5, 3));        // int
System.out.println(sumar(5.5, 3.2));    // double
Ventajas de los Métodos
  • Evitan repetición de código
  • Hacen el código más organizado
  • Facilitan el mantenimiento
  • Mejoran la legibilidad

6. Programación Orientada a Objetos (POO)

Java es un lenguaje completamente orientado a objetos, lo que significa que todo se basa en clases y objetos. Este enfoque permite organizar mejor el código y representar elementos del mundo real.

Conceptos Principales de POO

  • Encapsulamiento: Proteger los datos dentro de clases
  • Herencia: Reutilizar código mediante jerarquía de clases
  • Polimorfismo: Usar métodos de diferentes formas
  • Abstracción: Simplificar sistemas complejos

Definición de una Clase

public class Persona {
    // Atributos
    private String nombre;
    private int edad;
    
    // Método constructor
    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }
    
    // Métodos
    public void presentarse() {
        System.out.println("Me llamo " + nombre);
        System.out.println("Tengo " + edad + " años");
    }
}

Uso de la Clase

public class Main {
    public static void main(String[] args) {
        Persona p1 = new Persona("Ana", 25);
        p1.presentarse();
    }
}

Herencia

class Animal {
    public void hacerSonido() {
        System.out.println("Sonido genérico");
    }
}

class Perro extends Animal {
    @Override
    public void hacerSonido() {
        System.out.println("¡Guau!");
    }
}

class Gato extends Animal {
    @Override
    public void hacerSonido() {
        System.out.println("¡Miau!");
    }
}

public class Main {
    public static void main(String[] args) {
        Perro perro = new Perro();
        Gato gato = new Gato();
        
        perro.hacerSonido();  // ¡Guau!
        gato.hacerSonido();   // ¡Miau!
    }
}

Encapsulamiento con getters y setters

public class CuentaBancaria {
    private double saldo;
    
    public CuentaBancaria(double saldoInicial) {
        this.saldo = saldoInicial;
    }
    
    // Getter
    public double getSaldo() {
        return saldo;
    }
    
    // Setter con validación
    public void setSaldo(double nuevoSaldo) {
        if (nuevoSaldo >= 0) {
            this.saldo = nuevoSaldo;
        }
    }
}
La Importancia de POO

POO permite modelar el mundo real en código. Por ejemplo, una clase "Estudiante" puede tener atributos como nombre, edad, calificaciones y métodos como estudiar() o presentarExamen().

7. Manejo de Errores (Excepciones)

En Java, los errores se manejan mediante un sistema llamado excepciones. Esto permite controlar fallos sin que el programa se detenga inesperadamente.

Se utilizan bloques try-catch para capturar errores y manejarlos adecuadamente. También existe finally, que se ejecuta siempre, independientemente de si ocurre una excepción.

Estructura Básica

try {
    // Código que puede generar excepción
    int resultado = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("No se puede dividir entre cero");
}

Manejo de Múltiples Excepciones

try {
    int[] numeros = {1, 2, 3};
    System.out.println(numeros[5]);  // Índice fuera de rango
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("Índice no válido");
} catch (Exception e) {
    System.out.println("Error general");
}

Bloque finally

El bloque finally siempre se ejecuta, ocurra o no una excepción:

try {
    int resultado = 10 / 2;
    System.out.println("Resultado: " + resultado);
} catch (ArithmeticException e) {
    System.out.println("Error");
} finally {
    System.out.println("Operación finalizada");
}

Excepciones Comunes

Excepción Causa Ejemplo
ArithmeticException Error matemático División entre cero
ArrayIndexOutOfBoundsException Índice inválido array[100] en array de 10 elementos
NullPointerException Referencia nula Acceder a método en objeto null
NumberFormatException Formato de número inválido Integer.parseInt("abc")
Regla de Oro

Siempre usa try-catch para operaciones que pueden fallar (lectura de archivos, conexiones, conversiones de tipos).

8. Librerías y API de Java

Java cuenta con una amplia biblioteca estándar (API) que incluye herramientas para realizar muchas tareas sin necesidad de programarlas desde cero.

Estas librerías permiten trabajar con archivos, redes, interfaces gráficas, bases de datos y mucho más. Además, existen librerías externas que amplían aún más sus capacidades.

Librerías Estándar Comunes

java.util.Scanner - Entrada de Usuario

import java.util.Scanner;

public class Entrada {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        System.out.print("Ingresa tu nombre: ");
        String nombre = sc.nextLine();
        
        System.out.print("Ingresa tu edad: ");
        int edad = sc.nextInt();
        
        System.out.println("Hola " + nombre + ", tienes " + edad + " años");
    }
}

java.util.ArrayList - Listas Dinámicas

import java.util.ArrayList;

public class Listas {
    public static void main(String[] args) {
        ArrayList<String> frutas = new ArrayList<>();
        
        frutas.add("Manzana");
        frutas.add("Banana");
        frutas.add("Naranja");
        
        for (String fruta : frutas) {
            System.out.println(fruta);
        }
    }
}

java.io.File - Manejo de Archivos

import java.io.File;

public class Archivos {
    public static void main(String[] args) {
        File archivo = new File("datos.txt");
        
        if (archivo.exists()) {
            System.out.println("El archivo existe");
            System.out.println("Nombre: " + archivo.getName());
            System.out.println("Tamaño: " + archivo.length());
        }
    }
}

java.time - Fecha y Hora

import java.time.LocalDate;
import java.time.LocalDateTime;

public class Fechas {
    public static void main(String[] args) {
        LocalDate hoy = LocalDate.now();
        LocalDateTime ahora = LocalDateTime.now();
        
        System.out.println("Fecha: " + hoy);
        System.out.println("Fecha y Hora: " + ahora);
    }
}

9. Portabilidad (Write Once, Run Anywhere)

Una de las características más importantes de Java es su portabilidad. Gracias a la JVM, el código se convierte en bytecode, que puede ejecutarse en cualquier sistema que tenga instalada la máquina virtual.

Esto significa que no importa si el usuario usa Windows, Linux o Mac, el programa funcionará igual.

¿Cómo Funciona la Portabilidad?

Paso Proceso Resultado
1 Escribir código en .java Archivo de código fuente
2 Compilar con javac Archivo .class (bytecode)
3 JVM interpreta bytecode Ejecución en cualquier SO

Ventajas de la Portabilidad

  • Desarrolla una vez, ejecuta en cualquier lugar
  • No necesitas recompilar para diferentes sistemas
  • Facilita la distribución de aplicaciones
  • Reduce costos de desarrollo

Comparación: Java vs Otros Lenguajes

Lenguaje Windows Linux Mac
Java ✓ (con JVM) ✓ (con JVM) ✓ (con JVM)
C/C++ ✓ (recompilar) ✓ (recompilar) ✓ (recompilar)
Python ✓ (con Python) ✓ (con Python) ✓ (con Python)
WORA: Write Once, Run Anywhere

Este es el lema de Java que refleja perfectamente su capacidad de portabilidad. Es una de las razones principales de su éxito empresarial.

Plataformas Web

Software Empresarial

ERP, CRM y otros sistemas grandes utilizan Java.

Big Data y Analytics

Plataformas como Hadoop y Spark utilizan Java.
