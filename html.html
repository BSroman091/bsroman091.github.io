1. Estructura de Páginas Web

HTML (HyperText Markup Language) es el lenguaje fundamental para crear la estructura de cualquier página web. Imagina que estás construyendo una casa: HTML serían los cimientos, las paredes, el techo, las habitaciones; es decir, los elementos básicos que le dan forma y organización al contenido. Cada página web que visitas está construida utilizando HTML para definir dónde va el título, dónde se colocan los párrafos, dónde se insertan las imágenes, y cómo se organizan los diferentes bloques de información.

Definición: Página Web

Una página web es un documento electrónico que puede contener texto, imágenes, enlaces, videos y otros elementos multimedia, accesible a través de Internet mediante un navegador web.

Detalles y Ejemplos

HTML utiliza un sistema de etiquetas para marcar los diferentes componentes de una página. Por ejemplo, un título principal se marca con <h1>, un párrafo con <p>, y una lista con <ul> (para listas desordenadas) o <ol> (para listas ordenadas).

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Estructura Básica</title>
</head>
<body>
    <!-- Este es el contenido principal de la página -->
    <h1>Bienvenido a Mi Sitio Web</h1>
    <p>Este es el primer párrafo de introducción. Aquí explicamos de qué trata la página.</p>
    
    <h2>Sección de Contenido</h2>
    <p>En esta sección, presentamos información más detallada.</p>
    <ul>
        <li>Elemento de lista 1</li>
        <li>Elemento de lista 2</li>
    </ul>
</body>
</html>

En este ejemplo, <h1> define el título principal, <p> define párrafos, <h2> define un subtítulo, y <ul> con <li> crea una lista con viñetas. Todo esto contribuye a la estructura general y la legibilidad de la página para el usuario y para los motores de búsqueda.

2. Uso de Etiquetas (Elementos)

Las etiquetas son los bloques de construcción fundamentales de HTML. Son palabras clave encerradas entre corchetes angulares (< y >). La mayoría de las etiquetas vienen en pares: una etiqueta de apertura y una etiqueta de cierre. El contenido que se quiere marcar se coloca entre estas dos etiquetas. El conjunto de la etiqueta de apertura, el contenido y la etiqueta de cierre se conoce como un elemento HTML.

Definición: Elemento HTML

Un elemento HTML es un componente individual de una página web, definido por una etiqueta de apertura, contenido y una etiqueta de cierre (o en algunos casos, solo una etiqueta de apertura si es un elemento vacío).

Detalles y Ejemplos

  • Elementos de Bloque: Ocupan todo el ancho disponible y comienzan en una nueva línea (ej. <div>, <p>, <h1>, <ul>).
  • Elementos en Línea: Ocupan solo el ancho necesario y no comienzan en una nueva línea (ej. <span>, <a>, <strong>, <em>).
  • Elementos Vacíos: No tienen contenido y solo consisten en una etiqueta de apertura (ej. <img>, <br>, <hr>, <input>).
<!-- Ejemplo de un elemento de bloque (párrafo) -->
<p>Este es un párrafo de texto.</p>

<!-- Ejemplo de un elemento en línea (enlace) -->
Visita <a href="https://www.ejemplo.com">nuestro sitio</a> para más información.

<!-- Ejemplo de un elemento vacío (salto de línea) -->
Línea uno.<br>Línea dos.

<!-- Ejemplo de imagen (elemento vacío) -->
<img src="ruta/a/tu/imagen.jpg" alt="Descripción de la imagen">

La correcta utilización de las etiquetas es crucial para que el navegador interprete el contenido de la página de manera adecuada y para que los motores de búsqueda entiendan la estructura y el propósito de cada parte.

3. Atributos para Información Adicional

Los atributos proporcionan información extra o modifican el comportamiento de los elementos HTML. Se especifican dentro de la etiqueta de apertura y generalmente vienen en pares de nombre="valor". Los atributos son esenciales para personalizar y dar funcionalidad a los elementos.

Definición: Atributo HTML

Un atributo HTML es una propiedad adicional que se puede aplicar a un elemento HTML para modificar su comportamiento, apariencia o proporcionar metadatos.

Atributos Comunes y su Uso

href (en <a>)

Define la URL de destino de un enlace.

<a href="https://www.wikipedia.org">Ir a Wikipedia</a>

src (en <img>)

Especifica la ruta (URL o ruta de archivo) de la imagen a mostrar.

<img src="imagenes/logo.png" alt="Logo de la empresa">

alt (en <img>)

Proporciona un texto alternativo para la imagen. Es vital para la accesibilidad (lectores de pantalla) y si la imagen no se puede cargar.

<img src="gato.jpg" alt="Un gato atigrado durmiendo">

id

Identifica un elemento de forma única en la página. Útil para aplicar estilos CSS específicos o para ser manipulado por JavaScript.

<div id="seccion-principal">Contenido importante</div>

class

Permite agrupar varios elementos que comparten características. Se puede aplicar el mismo class a múltiples elementos. Muy usado para CSS y JavaScript.

<p class="destacado">Este texto es importante.</p>
<span class="destacado">Otro texto destacado.</span>

style

Permite aplicar estilos CSS directamente al elemento (CSS en línea). Aunque útil para ejemplos rápidos, generalmente se prefiere usar archivos CSS externos para mantener el código organizado.

<p style="color: blue; font-size: 16px;">Texto azul de 16px.</p>

Los atributos hacen que los elementos HTML sean mucho más potentes y adaptables a las necesidades del diseño y la funcionalidad web.

4. Jerarquía de Elementos (Árbol DOM)

Los elementos HTML en una página no están simplemente uno al lado del otro; están anidados unos dentro de otros, formando una estructura jerárquica. Esta estructura se conoce como el Árbol de Documento de Objeto (DOM - Document Object Model). El navegador interpreta esta jerarquía para entender la relación entre los elementos.

Definición: DOM (Document Object Model)

El DOM es una representación en memoria de un documento HTML o XML, estructurada como un árbol de nodos (elementos, atributos, texto). Permite a los lenguajes de scripting (como JavaScript) acceder y manipular el contenido, la estructura y el estilo del documento.

Estructura Jerárquica

En el árbol DOM:

  • El elemento raíz es <html>.
  • Los elementos dentro de <html> son head y body.
  • Los elementos dentro de body pueden ser otros elementos (<h1>, <p>, <div>, etc.).
  • Los elementos pueden contener otros elementos (elementos "padre" y "hijo") o texto.
<!DOCTYPE html>
<html> <!-- Nodo Raíz -->
  <head> <!-- Nodo Hijo de html -->
    <title>Ejemplo DOM</title> <!-- Nodo Hijo de head -->
  </head>
  <body> <!-- Nodo Hijo de html -->
    <h1>Título Principal</h1> <!-- Nodo Hijo de body -->
    <p> <!-- Nodo Hijo de body -->
      Este es un párrafo con un <span>elemento en línea</span>. <!-- span es hijo de p -->
    </p>
  </body>
</html>

Relaciones en el Árbol

  • <html> es el padre de <head> y <body>.
  • <head> es el padre de <title>.
  • <body> es el padre de <h1> y <p>.
  • <p> es el padre de un nodo de texto y del elemento <span>.
  • <span> es el padre de un nodo de texto.

Comprender esta jerarquía es fundamental para aplicar estilos CSS de manera efectiva (por ejemplo, aplicar un estilo a todos los <li> dentro de un <ul> específico) y para manipular elementos con JavaScript.

5. Semántica para Significado y Accesibilidad

La semántica en HTML se refiere al uso de etiquetas que describen el significado del contenido que encierran, en lugar de solo su apariencia. Usar etiquetas semánticas ayuda tanto a los navegadores como a los motores de búsqueda (SEO) a entender la estructura y el propósito del contenido, y es crucial para la accesibilidad web.

Definición: Semántica HTML

La semántica HTML es el uso de etiquetas HTML que comunican el significado inherente del contenido, más allá de su simple presentación visual.

Definición: Accesibilidad Web

La accesibilidad web asegura que las personas con discapacidades (visuales, auditivas, motoras, cognitivas) puedan percibir, entender, navegar e interactuar con la web.

Etiquetas Semánticas de HTML5

Antes, se usaban mucho <div> y <span> para todo. Ahora, HTML5 introduce etiquetas semánticas más específicas:

  • <header>: Contiene información introductoria o de navegación (logo, título principal, menú).
  • <nav>: Representa una sección de enlaces de navegación.
  • <main>: Especifica el contenido principal y único del documento. Debe haber solo uno por página.
  • <article>: Contenido independiente y autocontenido (ej. una publicación de blog, un artículo de noticias).
  • <section>: Una sección genérica temática dentro de un documento.
  • <aside>: Contenido relacionado tangencialmente con el contenido principal (ej. barras laterales, notas al pie).
  • <footer>: Contiene información del pie de página (autor, derechos de autor, enlaces de contacto).

Ejemplo de Estructura Semántica

<body>
  <header>
    <img src="logo.png" alt="Logo Empresa">
    <nav>
      <ul>
        <li><a href="#inicio">Inicio</a></li>
        <li><a href="#productos">Productos</a></li>
        <li><a href="#contacto">Contacto</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <h1>Título del Artículo Principal</h1>
      <p>Contenido del artículo...</p>
      <section>
        <h2>Subsección Relevante</h2>
        <p>Detalles de la subsección.</p>
      </section>
    </article>
    <aside>
      <h3>Publicidad</h3>
      <p>Anuncios relacionados.</p>
    </aside>
  </main>

  <footer>
    <p>© 2023 Mi Empresa. Todos los derechos reservados.</p>
  </footer>
</body>

El uso de estas etiquetas ayuda a los lectores de pantalla a interpretar la estructura de la página (por ejemplo, identificar rápidamente la navegación principal o el contenido principal) y a los motores de búsqueda a indexar mejor el contenido.

6. No es un Lenguaje de Programación

Es crucial entender que HTML no es un lenguaje de programación. Los lenguajes de programación (como JavaScript, Python, Java) permiten realizar operaciones lógicas, tomar decisiones (condicionales), repetir acciones (bucles) y manipular datos de forma dinámica. HTML, por otro lado, se enfoca exclusivamente en marcar y estructurar el contenido.

Definición: Lenguaje de Programación

Un lenguaje de programación es un conjunto formal de instrucciones y reglas sintácticas que permiten a un programador escribir código para controlar el comportamiento de una máquina (computadora).

Diferencia entre HTML y Lenguajes de Programación

  • HTML: Define qué es el contenido (un título, un párrafo, una imagen).
  • Lenguaje de Programación (ej. JavaScript): Define qué hacer con el contenido o cómo interactuar con él (ej. "cuando el usuario haga clic en este botón, muestra una alerta", "calcula la suma de estos dos números").

HTML se combina con lenguajes de programación como JavaScript para crear páginas web dinámicas e interactivas. CSS se usa para estilizar, HTML para estructurar, y JavaScript para la interactividad.

<!-- Esto es HTML - Define la estructura -->
<button id="miBoton">Haz clic aquí</button>
<p id="mensaje"></p>

<!-- Esto es JavaScript - Define la lógica/interactividad -->
<script>
  document.getElementById("miBoton").onclick = function() {
    document.getElementById("mensaje").innerText = "¡Has hecho clic!";
  };
</script>

En el ejemplo anterior, HTML crea el botón y el párrafo. JavaScript escucha el evento "clic" en el botón y, cuando ocurre, modifica el contenido del párrafo. HTML por sí solo no podría realizar esta acción interactiva.

7. Estándar Web Mantenido por W3C

HTML es un estándar web, lo que significa que hay un conjunto de reglas y especificaciones que definen cómo debe funcionar. Este estándar es desarrollado y mantenido por el World Wide Web Consortium (W3C), una organización internacional que establece las directrices para la web.

Definición: W3C (World Wide Web Consortium)

El W3C es la principal organización de estandarización de la World Wide Web. Desarrolla recomendaciones y directrices sobre tecnologías web, asegurando la interoperabilidad y la accesibilidad.

Importancia del Estándar W3C

El W3C publica diferentes especificaciones para las versiones de HTML (actualmente HTML5 es la versión más reciente y ampliamente adoptada). Estas especificaciones aseguran que:

  • Los navegadores web (Chrome, Firefox, Safari, Edge) interpreten el código HTML de manera consistente.
  • Las páginas web funcionen correctamente en diferentes dispositivos y plataformas.
  • Se promueva la accesibilidad y la usabilidad de la web.

La adhesión a los estándares del W3C ayuda a crear una web más robusta, predecible y accesible para todos. Cuando un nuevo elemento o atributo se introduce en HTML, pasa por un proceso de desarrollo y consenso dentro del W3C antes de ser adoptado como estándar.

8. Base para CSS y JavaScript

HTML, CSS y JavaScript son las tres tecnologías fundamentales del desarrollo web frontend. Cada una tiene un rol específico pero interdependiente:

  • HTML: Proporciona la estructura y el contenido semántico.
  • CSS: Se encarga de la presentación y el estilo visual (colores, fuentes, diseño, layout).
  • JavaScript: Añade interactividad y comportamiento dinámico a la página.
Definición: Frontend

El Frontend (o lado del cliente) se refiere a todo lo que el usuario ve e interactúa directamente en su navegador web.

Sinergia entre HTML, CSS y JavaScript

Sin HTML, no habría nada que estilizar con CSS ni nada con lo que interactuar con JavaScript. HTML es el lienzo sobre el cual trabajan las otras dos tecnologías.

<!DOCTYPE html>
<html>
<head>
    <title>HTML, CSS, JS</title>
    <!-- Enlace a un archivo CSS externo -->
    <link rel="stylesheet" href="estilos.css"> 
</head>
<body>
    <!-- HTML: Estructura -->
    <h1 id="titulo-principal">Mi Página Web</h1>
    <button>Haz clic</button>

    <!-- JavaScript: Lógica -->
    <script src="script.js"></script> 
</body>
</html>

Cómo Trabajan Juntas

  • estilos.css (archivo CSS) usaría el id="titulo-principal" para cambiar el color y tamaño de la fuente del <h1>.
  • script.js (archivo JavaScript) podría añadir una funcionalidad al botón, como mostrar un mensaje cuando se hace clic.

La sinergia entre estas tres tecnologías es lo que permite crear experiencias web ricas y complejas.

9. Tipos Comunes de Etiquetas

Existen cientos de etiquetas HTML, pero algunas son usadas con mucha más frecuencia y son esenciales para construir la mayoría de las páginas web. Conocerlas bien es un gran paso para dominar HTML.

Encabezados (<h1> a <h6>)

Definen títulos y subtítulos, organizando la jerarquía del contenido. <h1> es el más importante.

<h1>Título Principal</h1>
<h2>Sección Importante</h2>
<h3>Subsección</h3>

Párrafo (<p>)

Contiene bloques de texto.

<p>Este es un párrafo informativo.</p>

Enlace (<a>)

Crea hipervínculos a otras páginas o recursos. Requiere el atributo href.

<a href="otra_pagina.html">Ir a la otra página</a>

Imagen (<img>)

Inserta imágenes. Requiere src (fuente) y alt (texto alternativo). Es un elemento vacío.

<img src="logo.png" alt="Logo de la compañía">

Lista Desordenada (<ul>)

Crea listas con viñetas. Sus elementos hijos son <li>.

<ul>
    <li>Elemento 1</li>
    <li>Elemento 2</li>
    <li>Elemento 3</li>
</ul>

Lista Ordenada (<ol>)

Crea listas numeradas. También usa <li> para sus elementos.

<ol>
    <li>Primer paso</li>
    <li>Segundo paso</li>
    <li>Tercer paso</li>
</ol>

División (<div>)

Un contenedor genérico de bloque. Se usa para agrupar elementos y aplicarles estilos o scripts.

<div class="contenedor">
    <p>Contenido dentro de la división</p>
</div>

Span (<span>)

Un contenedor genérico de línea. Se usa para estilizar o manipular una parte pequeña de texto.

<p>Este es un <span style="color: red;">texto rojo</span> dentro de un párrafo.</p>

Formulario (<form>, <input>, <button>)

Se usan para crear formularios interactivos donde los usuarios pueden ingresar información.

<form>
    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre">
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email">
    
    <button type="submit">Enviar</button>
</form>

Tabla (<table>, <tr>, <td>, <th>)

Se usan para presentar datos de forma tabular.

<table>
    <tr>
        <th>Nombre</th>
        <th>Edad</th>
    </tr>
    <tr>
        <td>Juan</td>
        <td>25</td>
    </tr>
    <tr>
        <td>María</td>
        <td>30</td>
    </tr>
</table>
